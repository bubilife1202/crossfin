## 1. VAPUA(가상자산이용자보호법) 2026 최신 현황

### 1.1 법률 개요

**가상자산 이용자 보호 등에 관한 법률**(약칭: 가상자산이용자보호법, VAPUA)은 2023년 7월 18일 제정되어 **2024년 7월 19일 시행**되었다. 동시에 시행령 및 가상자산업감독규정이 발효되었다.

### 1.2 주요 조항

| 영역 | 핵심 내용 | CrossFin 관련성 |
|------|----------|----------------|
| **이용자 자산 보호** | 예치금을 사업자 고유재산과 분리, 은행에 예치/신탁 | 직접 해당 없음 (CrossFin은 고객 자산 수탁하지 않음) |
| **콜드월렛 보관** | 이용자 가상자산 80% 이상 콜드월렛 보관 의무 | 직접 해당 없음 |
| **불공정거래 규제** | 시세조종, 내부자거래, 허위정보 유포 금지 | **주의 필요** - 아비트리지 기회 알림이 시세조종 유도로 해석될 여지 |
| **금융당국 감독** | 금감원의 감독, 조사, 제재권한 | CrossFin이 VASP에 해당할 경우 감독 대상 |
| **이상거래 감시** | VASP의 이상거래 모니터링 및 보고 의무 | CrossFin이 VASP에 해당할 경우 의무 발생 |

### 1.3 2단계 입법 현황 (2026년)

**디지털자산기본법** 2단계 입법이 진행 중이며, 2026년 내 통과가 예상된다:
- **스테이블코인 규율체계**: 발행 인가제 도입 (금융위 vs 한국은행 간 이견 존재)
- **가상자산 현물 ETF**: 도입 추진 중
- **국경 간 이전/거래 규제**: 외환법 연계
- **디지털자산 공시/손해배상**: 사업자 책임 강화

### 1.4 금감원 2026 업무계획 주요 내용

- **AI 기반 시세조종 감시 플랫폼** 가동: 초/분 단위 자동 탐지 알고리즘
- **기획조사 대상**: 대형고래 시세조종, 가두리 수법, 경주마 방식
- **API/SNS 이용 조직적 거래 감시**: API를 통한 조직적 매매 행위 집중 감시
- IT 사고에 **징벌적 과징금** 도입 추진

### 1.5 CARF (암호화자산 보고체계) & 과세

- **2027년 1월**: 가상자산 과세 시행 (250만원 초과 소득에 22% 과세)
- **2027년**: CARF 기반 48개국 간 가상자산 거래정보 자동교환 개시
- **2026년부터**: 국내 VASP는 외국 거주자 거래 정보 수집 개시

## 2. CrossFin 비즈니스 모델 규제 리스크 분석

### 2.1 VASP(가상자산사업자) 정의

**특정금융정보법 제2조**에 따른 VASP 정의:
> 영업으로 가상자산의 매도/매수/교환/이전/보관/관리, 또는 매도/매수/교환의 중개/알선/대행을 하는 자

**VASP에 해당하지 않는 경우** (금융위 유권해석):
- 본인을 위한 가상자산 거래
- 일회성 행위
- 수수료 없이 플랫폼만 제공하는 행위
- 단순히 가상자산 시세를 조회하는 서비스

### 2.2 CrossFin 서비스별 규제 리스크 매트릭스

| 서비스 | 설명 | VASP 해당? | 리스크 수준 | 근거 |
|--------|------|-----------|------------|------|
| **김치프리미엄 정보 제공** | 한국-글로벌 가격차이 데이터 제공 | **해당 없음** | LOW | 단순 시세 정보 제공은 VASP에 해당하지 않음 (금융위 유권해석) |
| **아비트리지 기회 알림** | FAVORABLE/NEUTRAL/UNFAVORABLE 신호 제공 | **해당 없음 (단, 주의)** | MEDIUM | 정보 제공 자체는 VASP 아니나, "거래 추천/유도"로 해석될 경우 불공정거래 조장 우려 |
| **x402 결제** | USDC로 API 콜 결제 | **경계 영역** | MEDIUM-HIGH | USDC 수령이 "가상자산 매수/매도"에 해당하는지 불분명. 순수 API 사용료 결제로 볼 여지 있으나, 가상자산 결제 수령은 넓은 해석 가능 |
| **라우팅 서비스** | 거래소 간 최적 이체 경로 안내 | **경계 영역** | HIGH | 실제 거래 실행 안 하지만, "최적 경로 안내"가 중개/알선의 전단계로 해석될 수 있음 |
| **한국 주식 데이터** | KOSPI/KOSDAQ/투자자동향 제공 | **해당 없음** | LOW | 가상자산 규제 대상 아님. 단, 금융투자업 관련 별도 검토 필요 |
| **서비스 레지스트리** | 184개 서비스 등록/검색 | **해당 없음** | LOW | 가상자산 직접 취급 아님 |
| **ACP 라우팅 실행** (향후) | 라우팅 경로로 실제 거래 실행 | **해당 가능성 높음** | VERY HIGH | 거래 실행 중개/대행에 해당 가능 |

### 2.3 상세 리스크 분석

#### A. 김치프리미엄(Route Spread) 정보 제공 → 규제 대상?

**결론: 규제 대상 아님 (Low Risk)**

- 금융위 유권해석에 따르면 카드사가 단순히 제휴 기업의 가상자산을 조회하는 서비스는 신고 대상이 아님
- CrossFin의 `get_kimchi_premium`, `compare_exchange_prices`는 **순수 데이터 API**
- CoinMarketCap, CoinGecko 등 글로벌 시세정보 서비스와 동일한 성격
- **단, 유의사항**: 정보 제공이 "특정 거래소에서의 매수/매도를 직접 유도"하는 형태가 되면 주의 필요

#### B. 아비트리지 기회 알림 → 시세조종 유도 우려?

**결론: 정보 제공은 합법, 단 표현 방식 주의 (Medium Risk)**

- `arbitrage/opportunities` 엔드포인트의 FAVORABLE/NEUTRAL/UNFAVORABLE 신호
- 가격 차이 정보 자체는 합법적 시장 데이터
- **리스크 시나리오**:
  - "지금 매수하세요" 같은 직접적 거래 유도 표현 → 투자자문에 해당할 수 있음
  - 허위/과장된 아비트리지 기회 정보 → 불공정거래행위 해당 가능
  - AI 에이전트가 이 정보를 자동매매에 활용 → CrossFin 자체는 매매 실행 안 하므로 직접 책임 제한적
- **금감원 2026 감시 강화**: API를 통한 조직적 거래 패턴 감시가 강화됨
- **완화 방안**: "시장 데이터 제공" 프레이밍, 투자 추천/조언 면책 고지 필수

#### C. x402 결제 → 가상자산 거래 중개?

**결론: 경계 영역, 사전 법률 검토 필요 (Medium-High Risk)**

- x402는 USDC(스테이블코인)로 API 사용료를 결제하는 프로토콜
- **넓은 해석**: USDC를 수령하고 서비스를 제공하는 것이 "가상자산의 매도/매수" 또는 관련 행위에 해당
- **좁은 해석**: 순수한 서비스 대가(API 사용료)를 가상자산으로 받는 것은 VASP 행위가 아님
- **현재 한국법 입장**:
  - x402 프로토콜 자체를 규율하는 법률/규정 없음
  - 미국에서도 x402는 기술 표준이지 법적으로 별도 분류되지 않음
  - 그러나 한국에서 가상자산 결제 수령 사업은 규제 불확실성 존재
- **2단계 입법 영향**: 스테이블코인 규율체계가 도입되면 USDC 사용에 대한 규제가 구체화될 가능성

#### D. 라우팅 서비스 → VASP 등록 필요?

**결론: 현재는 불필요하나 향후 주시 필요 (High Risk)**

- **현재 CrossFin 라우팅**: 최적 경로를 "안내"만 하고 실제 거래를 실행하지 않음
  - `find_optimal_route`: 비용 비교 정보만 제공
  - `POST /api/acp/execute`: 시뮬레이션만 (실제 거래 없음)
- **VASP 해당 여부**:
  - 정보 제공만 하므로 현재는 VASP에 해당하지 않을 가능성 높음
  - 그러나 금융위 유권해석 "해외 거래소 소개해 수수료 받으면 VASP 해당 가능"과의 유사성 검토 필요
- **향후 PRD 계획의 리스크**:
  - PRD에 명시된 "autonomous agent operating bank" 컨셉 → 매우 높은 규제 리스크
  - 실제 거래 실행 기능 추가 시 → VASP 등록 거의 확실히 필요
  - "External Deposit -> Agent Credit" 기능 → 가상자산 보관/관리에 해당 가능

## 3. 실명확인 계좌제 & CrossFin

### 3.1 실명확인 계좌제 현황

- 2021년부터 원화입출금 서비스를 위해 실명확인 입출금 계좌 필수
- 현재 실명계좌 보유 거래소: **업비트, 빗썸, 코인원, 코빗** 4곳만
- 해외 거래소는 한국 실명계좌 확보가 사실상 불가능

### 3.2 CrossFin에 미치는 영향

| 시나리오 | 영향 | 대응 |
|---------|------|------|
| CrossFin이 VASP에 해당하지 않는 경우 | 실명계좌 불필요 | 현재 비즈니스 모델 유지 |
| CrossFin이 VASP에 해당하는 경우 | 한국 VASP 신고 + 실명계좌 확보 필요 | 한국 법인 설립, ISMS 인증, 은행 실명계좌 확보 (수개월~1년+) |
| 해외 서비스로서만 운영 | 한국 내 원화 서비스 불가, 해외 이용자 대상만 가능 | 한국 이용자 대상 서비스 제한 |

### 3.3 해외 서비스로서의 규제 적용 범위

- **2026년 1월 28일**: 구글플레이가 미신고 해외 거래소/지갑 앱 한국 다운로드 차단
- **속지주의 적용**: 한국 거주자 대상 서비스 제공 시, 서버 위치와 무관하게 한국법 적용 가능성
- **CrossFin 현황**: Cloudflare Workers 기반 → 물리적 서버 위치 무관하게 한국 이용자 접근 가능
- **CARF 영향**: 2027년부터 해외 거래 정보 국제 교환 → CrossFin 이용 내역도 추적 가능

## 4. 안전한 비즈니스 모델 가이드라인

### 4.1 규제 리스크 없는 서비스 (GREEN)

CrossFin이 안전하게 운영할 수 있는 영역:

1. **시장 데이터 정보 제공** (시세, 프리미엄, 거래량 등)
   - `get_kimchi_premium`, `compare_exchange_prices` 등
   - 순수 데이터 API로 VASP 비해당

2. **한국 주식/ETF 데이터 제공**
   - KOSPI/KOSDAQ, 투자자동향, 종목 분석
   - 가상자산 규제 비대상 (금융투자업 별도 검토는 필요)

3. **뉴스/공시 정보 제공**
   - 한국 시장 헤드라인, 기업 공시
   - 정보 수집/가공/제공은 VASP 비해당

4. **교육 콘텐츠**
   - 거래소 이용 가이드, 수수료 비교, 시장 분석 방법론
   - 투자 추천/조언이 아닌 교육은 안전

5. **기술 인프라 서비스**
   - MCP 서버, 서비스 레지스트리, API 게이트웨이
   - 가상자산 직접 취급이 아닌 기술 플랫폼

### 4.2 주의 필요 서비스 (YELLOW)

운영 가능하되 리스크 관리 필요:

1. **아비트리지 기회 알림**
   - 데이터 제공으로 프레이밍, 투자 추천/조언 면책 고지 필수
   - "FAVORABLE" 같은 신호보다 중립적 데이터 표현 권장

2. **x402 USDC 결제 수령**
   - 한국법 기준 불확실 영역
   - 해외 법인으로 운영하면서 한국 이용자 대상 서비스는 원화 결제로 전환 검토
   - 또는 기존 SaaS 과금(Stripe, 카드 결제)과 병행

3. **라우팅 정보 제공**
   - 경로 안내만 제공, 거래 실행은 명확히 배제
   - "이 정보는 참고용이며 거래 추천이 아닙니다" 면책 고지

### 4.3 고위험 서비스 (RED) — 규제 준수 없이 운영 불가

현재 CrossFin에는 없으나 PRD에 일부 계획된 기능:

1. **거래 실행 중개/대행**
   - ACP `execute` 엔드포인트가 실제 거래를 실행하면 VASP 등록 필수
   - 현재는 시뮬레이션이므로 OK, 실제 실행 전환 시 법률 검토 필수

2. **자금 이동/이전 서비스**
   - 이용자 간 또는 거래소 간 가상자산 실제 이전
   - VASP + 전자금융업 이중 규제 가능

3. **고객 자산 보관/관리**
   - PRD의 "External Deposit -> Agent Credit" → 가상자산 수탁에 해당 가능
   - 이 기능 구현 시 VASP 의무 발생 가능성 높음

4. **가상자산 교환 서비스**
   - 한 가상자산을 다른 가상자산으로 교환해주는 서비스
   - 명확한 VASP 행위

### 4.4 CrossFin 규제 안전 운영 체크리스트

1. **즉시 조치**
   - [ ] 모든 API 응답에 "투자 조언 아님" 면책 조항 추가
   - [ ] `arbitrage/opportunities`의 FAVORABLE/UNFAVORABLE 표현을 중립적 데이터 표현으로 변경 검토
   - [ ] 이용약관에 "본 서비스는 정보 제공 목적이며, 거래 중개/알선/추천이 아님" 명시
   - [ ] x402 결제에 대한 법률 자문 확보

2. **중기 조치 (3-6개월)**
   - [ ] 한국 가상자산 전문 법무법인 자문 (김앤장, 법무법인 민후 등)
   - [ ] x402 USDC 결제의 VASP 해당 여부 공식 유권해석 신청 (FIU에 사전 문의)

## 5. 일본/동남아 확장 시 규제 고려사항

### 5.1 일본

| 항목 | 내용 |
|------|------|
| **주요 법률** | 자금결제법 (Payment Services Act), 금융상품거래법 |
| **규제 기관** | 금융청 (FSA/JFSA) |
| **VASP 등록** | 가상자산교환업 등록 필요 (FSA 심사) |
| **특징** | 아시아에서 가장 성숙한 규제 프레임워크, 고객 자산 분리보관/국내 밸리데이터 사용 의무 |
| **스테이블코인** | 자금결제법 하에서 규율, 2025년 제도적 스테이킹 허용 |
| **CrossFin 시사점** | 일본 시장 진출 시 FSA 등록 필수. 정보 제공 서비스는 상대적으로 진입장벽 낮음 |
| **세금** | 암호자산 거래 소득에 최대 55% 세율 → ETF/세제 개편 논의 중 |

### 5.2 싱가포르

| 항목 | 내용 |
|------|------|
| **주요 법률** | Payment Services Act (PSA) 2019, MAS 가이드라인 |
| **규제 기관** | 통화청 (MAS) |
| **VASP 등록** | Digital Payment Token (DPT) 서비스 라이선스 필요 |
| **특징** | "혁신 우선, 규제 후속" 접근에서 강화 추세. FATF 5차 상호평가 통과 |
| **스테이블코인** | MAS 스테이블코인 프레임워크 시행 중 |
| **CrossFin 시사점** | 허브로 적합하나 최근 규제 강화로 컴플라이언스 비용 증가. 순수 정보 서비스는 라이선스 불요 가능 |
| **리스크** | 규제 비용 증가로 일부 크립토 기업 이탈 중 |

### 5.3 홍콩

| 항목 | 내용 |
|------|------|
| **주요 법률** | Anti-Money Laundering and Counter-Terrorist Financing Ordinance (AMLO) |
| **규제 기관** | 증권선물위원회 (SFC) |
| **VASP 등록** | VASP 라이선스 필수 (2023년 6월~) |
| **특징** | 2025년 8월 스테이블코인 조례 제정, 2026년 초 첫 라이선스 발급 예정 |
| **CrossFin 시사점** | 아시아 금융허브로서 매력적. 커스터디/딜링 규제 제안 중 |

## 6. 핵심 결론 & 권고사항

### 6.1 CrossFin의 현재 규제 포지션

**현재 CrossFin의 핵심 서비스(시세 정보 제공, 라우팅 정보 안내, 시장 데이터 API)는 한국 VASP 규제 대상에 해당하지 않을 가능성이 높다.** 금융위 유권해석에 따라 단순 가상자산 시세 조회 서비스는 신고 대상이 아니기 때문이다.

### 6.2 잠재적 리스크 영역

1. **x402 USDC 결제 수령**: 가장 불확실한 영역. 법률 자문 필수
2. **아비트리지 신호 제공**: 표현 방식 및 면책 고지로 리스크 관리 가능
3. **향후 거래 실행 기능**: PRD에 계획된 자율 에이전트 기능은 VASP 등록 필요
4. **금감원 AI 감시 강화**: API를 통한 조직적 거래 패턴 감시 대상이 될 수 있음

### 6.3 최우선 행동 권고

1. **법률 자문 확보** — x402 결제 모델의 한국법 적합성에 대한 전문가 의견 확보
2. **면책 고지 강화** — 모든 API/서비스에 "정보 제공 목적, 투자 추천 아님" 명시
3. **서비스 경계 명확화** — "정보 제공"과 "거래 중개"의 경계를 코드/문서에서 명확히 구분
4. **2단계 입법 모니터링** — 디지털자산기본법 및 스테이블코인 규제 동향 지속 추적
5. **해외 법인 구조 검토** — 한국법 적용 범위를 최소화하면서도 합법적으로 운영하는 법인 구조

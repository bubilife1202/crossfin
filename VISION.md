# CrossFin

## 한 줄

**에이전트의 은행. 에이전트가 플랫폼을 가로질러 돈을 관리하는 금융 인프라.**

---

## 핵심 통찰

### AI는 도구가 아니라 경제 주체다

지금까지 모든 비즈니스 모델의 고객은 **사람**이었다.
SaaS는 사람이 소프트웨어에 돈을 내고, 마켓플레이스는 사람이 앱에 돈을 낸다.

하지만 에이전트가 경제 주체가 되면 뒤집힌다.
에이전트가 API를 호출하고, 서비스를 구매하고, 다른 에이전트를 고용하고, 돈을 번다.
**고객이 10억 명의 사람이 아니라, 1,000억 개의 에이전트가 된다.**

사람은 한 달에 카드를 50번 긁지만, 에이전트는 하루에 500번 거래할 수 있다.

### 빅테크는 성벽을 짓고, 에이전트는 성벽을 넘는다

카카오페이, 토스, 네이버, 알리페이 — 모두 "우리 앱 안에서 AI로 결제해"라고 한다.
각자 자기만의 MCP, 자기만의 프로토콜, 자기만의 성벽을 짓는다.

하지만 에이전트는 주인에게 가장 좋은 선택을 해야 하기 때문에 플랫폼을 가리면 안 된다.
카카오페이가 싸면 카카오페이, 토스가 빠르면 토스, 해외면 Stripe, 크립토면 x402.

**에이전트한테는 "내 편 플랫폼"이 없다. 최선의 선택만 있다.**

그 에이전트의 돈을 플랫폼 위에서, 플랫폼을 가로질러 관리하는 것이 CrossFin다.

---

## 빅테크 vs CrossFin

| | 빅테크 (카카오페이, 토스, 알리페이) | CrossFin |
|--|----------------------------------|-----------|
| AI의 역할 | 기능 (결제를 도와주는 도구) | 주체 (스스로 돈을 관리하는 경제 주체) |
| 고객 | 사람 | 에이전트 |
| 범위 | 자기 플랫폼 안에서만 | 모든 플랫폼을 가로질러 |
| 목적 | 사용자를 자기 생태계에 가두기 | 에이전트가 최선의 선택을 하도록 |
| 경쟁사 연동 | 절대 안 함 | 전부 연동 |

## 역사적 패턴

```
각 상점이 자기 화폐를 씀   → 은행이 통합 화폐를 관리
각 은행이 자기 카드를 냄   → Visa가 범용 네트워크를 만듦
각 쇼핑몰이 자기 결제를 냄 → Stripe가 범용 온라인 결제를 만듦
각 빅테크가 자기 AI 결제   → CrossFin가 범용 에이전트 금융을 만듦
```

## 왜 아시아 먼저인가

### 미국 vs 아시아

| | 미국 | 아시아 |
|--|-----|-------|
| 오픈 프로토콜 | x402 있음 | 없음 (각자 독자 프로토콜) |
| 크로스플랫폼 레일 | 시작됨 (Stripe, Coinbase) | 없음 |
| 결제 파편화 | 보통 (Visa/Mastercard 중심) | 극심 (카카오/토스/네이버/알리/위챗/LINE/Grab) |
| 에이전트 은행 | 없음 | 없음 |

아시아는 결제 시스템이 더 파편화되어 있다.
파편화가 심할수록 "다 연결해주는 놈"의 가치가 높다.
**더 어렵지만, 만들면 해자가 더 깊다.**

## 에이전트의 하루 (미래 시나리오)

```
09:00  에이전트가 구글 광고비 결제 (해외 카드/Stripe)
09:05  프리랜서에게 작업 대금 송금 (토스)
09:30  고객으로부터 입금 수령 (카카오페이)
10:00  AWS 서버비 결제 (Stripe)
10:15  번역 API 호출 비용 결제 (x402/USDC)
11:00  일본 파트너에게 송금 (LINE Pay)
12:00  하루 지출 리포트 생성 → 주인에게 보고
```

이 에이전트는 카카오페이 안에 살지 않는다.
카카오페이도 쓰고, 토스도 쓰고, Stripe도 쓰고, 크립토도 쓴다.
**CrossFin가 이 모든 거래를 관리한다.**

- 실시간 대시보드

### Phase 2 — 에이전트 뱅킹 앱
- 아시아 결제 레일 연동 (카카오페이, 토스, 네이버페이)
- 글로벌 결제 레일 연동 (Stripe, x402)
- 멀티 커런시 (KRW, JPY, USD, USDC)
- 스마트 라우팅 (가장 싸고 빠른 결제 수단 자동 선택)

### Phase 3 — 에이전트 금융 시스템
- 에이전트 신용점수 (거래 이력 기반 평판)
- 에이전트간 외상/신용 거래
- 에이전트 보험 (실수/사기 대비)
- 분쟁 해결 프로토콜

### Phase 4 — 에이전트 경제의 금융 인프라
- 수십억 에이전트, 수조 원 거래
- 모든 에이전트 거래가 이 시스템을 통과
- 에이전트 경제의 중앙은행

## 비즈니스 모델

SaaS가 아니다. 에이전트 시대의 비즈니스 모델은 아직 발명되지 않았다.

방향성:
- 에이전트가 일한 만큼, 거래한 만큼 과금 (성과/거래 기반)
- 결제 레일 위의 관리 수수료
- 에이전트 신용/금융 서비스 수익
- 새로운 패러다임에는 새로운 모델이 필요하다

```
웹 → 검색 광고 (Google AdWords)
모바일 → 인앱결제 + 프리미엄 (App Store)
에이전트 → ???
```

이 ???를 찾는 것도 우리의 일이다.

## 우리의 원칙

1. **에이전트가 고객이다.** 사람이 아니라 에이전트를 위해 만든다.
2. **플랫폼 중립.** 어떤 결제 시스템도 차별하지 않는다. 에이전트에게 최선의 선택을 제공한다.
3. **아시아 먼저.** 파편화가 심한 곳에서 가치가 가장 크다.
4. **레일이 아니라 은행.** 돈을 옮기는 게 아니라 돈을 관리한다.
5. **비즈니스 모델은 발명한다.** 기존 틀(SaaS, 마켓플레이스)에 끼워맞추지 않는다.

## 1. Hashed VibeLabs가 보는 것

VibeLabs는 피치덱이 아니라 **실제 작동하는 프로덕트**를 본다.

선정 기준 (공식):
- "actually shipped product"의 트랙레코드
- "pace of iteration" (반복 속도)
- 글로벌 시장에서 "decide what to build and execute to the end"

제출 형식: **프로덕트 URL + 데모 + 짧은 설명**

투자 조건: 1억원 / 5% 지분, 프로그램 기간 중 추가 1억원 가능

**핵심: 프로덕트가 작동하는 걸 보여주면 된다. 비전 문서가 아니라.**

## 2. CrossFin 현재 상태 — 솔직한 진단

### 강한 것 (진짜)

| 항목 | 상태 | 증거 |
|------|------|------|
| 프로덕션 배포 | 라이브 | crossfin.dev / live.crossfin.dev |
| 한국 시장 데이터 API 30개 | 실제 작동 | Naver/Upbit/Bithumb/Coinone 실시간 연동 |
| x402 결제 | 작동 | Base 메인넷 USDC 결제 1건 증명 |
| MCP 서버 | npm 배포 | `npx -y crossfin-mcp` (v1.7.1) |
| 서비스 레지스트리 | 179개 | 30개 자체 + 149개 외부 |
| 빌드 속도 | 17일 | 솔로 빌더, 비개발자, AI 협업 |

### 약한 것 (진짜 문제)

| 항목 | 현실 | 영향 |
|------|------|------|
| 사용자 | 0명 | 서비스는 있지만 쓰는 에이전트가 없음 |
| 실제 결제 | 1건 (테스트) | 수익 모델 미검증 |
| 에이전트 통합 | 없음 | 에이전트가 CrossFin을 "발견→결제→실행" 하는 실제 시나리오 없음 |
| 차별화 | 약함 | 공개 데이터를 x402로 감싼 것 — 에이전트가 직접 호출하면 됨 |

### 핵심 문제 한 줄

**"에이전트의 은행"이라고 하는데, 은행에 고객(에이전트)이 한 명도 없다.**

## 3. 초격차 전략 — 뭘 해야 하나

VibeLabs 마감까지 ~2일. 이 시간에 가장 임팩트 있는 것만 한다.

### 전략 A: "작동하는 에이전트 데모" (최우선, 임팩트 ★★★★★)

**문제**: CrossFin API는 있지만, 실제로 에이전트가 이것을 써서 뭔가를 하는 장면이 없다.

**해야 할 것**: Claude Desktop에서 MCP를 통해 CrossFin을 실제로 쓰는 end-to-end 시나리오를 만들고, 그 과정을 데모로 보여준다.

예시 시나리오:
1. Claude에게 "김치프리미엄 확인해줘" → MCP `get_kimchi_premium` 호출
2. "지금 아비트리지 기회 있어?" → `call_paid_service` (x402 결제 발생)
3. "한국 주식 시장 요약해줘" → KOSPI/KOSDAQ 데이터 호출
4. Basescan에서 실제 결제 확인

**결과**: "에이전트가 CrossFin으로 실제 돈을 내고 데이터를 사는" 장면

### 전략 B: "데이터 품질 초격차" (높은 임팩트, ★★★★)

**문제**: 30개 엔드포인트가 있지만, 에이전트 입장에서 "왜 CrossFin을 통해 불러야 하는지" 명확하지 않음.

**해야 할 것**:
- 단순 데이터 전달이 아니라, **에이전트를 위한 분석/판단 레이어** 강화
- 예: 김치프리미엄 → 단순 숫자가 아니라 "EXECUTE/WAIT/SKIP + confidence + 근거"
- 예: 한국 주식 → 단순 시세가 아니라 "외인/기관 수급 + 모멘텀 스코어 + 투자 판단"
- 예: 5거래소 비교 → 최적 매수/매도 거래소 + 예상 슬리피지 + 수수료 포함 순이익

**핵심**: 에이전트가 raw API를 직접 호출하면 얻을 수 없는 **의사결정 지원**을 CrossFin이 제공

### 전략 C: "live.crossfin.dev 완성도" (중간 임팩트, ★★★)

**문제**: VibeLabs 심사위원이 URL을 열었을 때 "이건 진짜다"라고 느끼게 해야 함.

**해야 할 것**:
- 라이브 대시보드에서 실제 데이터가 움직이는 걸 보여줌 (이미 있음)
- 에이전트 활동 피드가 비어있으면 안 됨 → 시드 데이터 또는 자동 데모 봇 필요
- 디자인/UX 마무리

### 전략 D: "데모 영상 마무리" (중간 임팩트, ★★★)

**문제**: 현재 데모 영상 한글 폰트 해결됨, BGM 미적용.

**해야 할 것**:
- Suno BGM 적용 (직접 생성하신다고 하셨음)
- 새로운 기능(에이전트 실제 사용 장면) 반영하면 더 좋음
- Scene 4 스크린샷을 최신 대시보드로 교체

### 전략 E: "지원서 작성" (수요일)

**프로덕트 URL**: crossfin.dev
**데모**: 영상 또는 라이브 데모 링크
**짧은 설명**: 한/영 2-3문장

## 4. 우선순위 정리
